
## **1. Introduction à Next.js 13+ et l'App Router**  
### 🎯 Objectifs : Comprendre les bases de Next.js et les différences avec les versions précédentes.  
- Présentation de Next.js : framework React orienté full-stack.  
- Différences entre `pages/` et `app/` (App Router vs Pages Router).  
- Avantages de l’architecture `app/` (Server Components, meilleure performance).  
- Installation et configuration d’un projet Next.js (`create-next-app`).  

📌 **Exercice** : Installer un projet Next.js 13+ et explorer la structure `app/`.  

---

## **2. Gestion des routes et des layouts**  
### 🎯 Objectifs : Comprendre la structure de routing de Next.js avec `appDir`.  
- Routing basé sur le système de fichiers (`app/` vs `pages/`).  
- Mise en place de **layouts** (`layout.tsx`) pour une structuration globale.  
- Utilisation des **loading states** (`loading.tsx`).  
- Gestion des erreurs (`error.tsx`).  
- Concept des **Server Components** vs **Client Components** (`'use client'`).  

📌 **Exercice** :  
1. Créer un layout global (`layout.tsx`) avec un header et un footer.  
2. Ajouter une page `about.tsx` et observer le comportement du routing.  
3. Implémenter une page avec un `loading.tsx` simulant un chargement asynchrone.  

---

## **3. Métadonnées et SEO dans Next.js 13+**  
### 🎯 Objectifs : Améliorer le référencement et le partage des pages.  
- Utilisation de `metadata` dans un `layout.tsx` ou une page.  
- Ajout de métadonnées dynamiques (`generateMetadata`).  
- Gestion des Open Graph & Twitter Cards.  
- Gestion des `robots.txt` et `sitemap.xml`.  

📌 **Exercice** :  
1. Ajouter un titre et une description dynamique à une page (`generateMetadata`).  
2. Configurer une image Open Graph pour une page spécifique.  

---

## **4. Server Actions et gestion des données**  
### 🎯 Objectifs : Exploiter les **Server Actions** pour améliorer la gestion des données.  
- Principe des Server Actions (remplacement des API routes).  
- Création d’une Server Action pour gérer un formulaire.  
- Utilisation avec **React Server Components**.  
- Gestion des mutations (`useOptimistic`).  

📌 **Exercice** :  
1. Créer un formulaire de contact avec une Server Action.  
2. Implémenter un bouton "Like" avec une mise à jour en temps réel (optimistic UI).
